1
Schritt 1 von 5
Dein Prompt — die Eingabe

Je klarer dein Prompt, desto besser die Antwort. Das ist der wichtigste Moment im gesamten Prozess.

✗ Schlechter Prompt
„Sag mir was über KI"

Problem: Kein Thema, kein Kontext, kein Ziel. Das Modell weiß nicht, was du brauchst.

✓ Guter Prompt
„Erkläre ISO 25010 Qualitätskriterien für KI-Systeme, einfach erklärt, mit Beispielen"

Warum besser: Konkretes Thema, klares Ziel, gewünschtes Format angegeben.

„Wer Anforderungen beherrscht, kann auch besser prompten."
Beide folgen derselben Logik: Präzision ist nicht optional.

Finale Antwort (Beispiel)

ISO/IEC 25010:2023 definiert neun Qualitätsmerkmale für Software und KI-Systeme: Functional Suitability, Performance Efficiency, Compatibility, Interaction Capability, Reliability, Security, Maintainability, Flexibility und Portability. Für KI ist besonders Explainability (Erklärbarkeit, unter Interaction Capability) und Autonomy wichtig — das System muss begründen, wie es zu seiner Antwort gelangt. Status: in Kraft seit Februar 2023.

Quellen: iso.org · bsi-group.com · enisa.europa.eu

Vier Leitplanken auf jedem Schritt

Diese Frameworks begleiten den gesamten Prozess — von der Eingabe bis zur Antwort.

ISO/IEC 25010:2023
Software-Qualität (9 Merkmale, Feb. 2023)

ISO/IEC 25010:2023 definiert 9 Qualitätsmerkmale — neu: Interaction Capability (inkl. Explainability) und Flexibility. In Kraft seit Februar 2023.

  • Functional Suitability: Antworten korrekt und vollständig
  • Performance Efficiency: effizient, ohne unnötige Umwege
  • Interaction Capability / Explainability: KI muss Entscheidungen begründen können
  • Flexibility (neu): System ohne Code-Änderung auf anderer Infrastruktur lauffähig
ISO 27001
Informationssicherheit

Schützt alle Daten auf dem Weg vom Nutzer ins Web und zurück.

  • Suchanfragen werden verschlüsselt übertragen (HTTPS)
  • Vertrauliche Eingaben verlassen nicht den sicheren Bereich
  • Zugriff auf externe Quellen wird protokolliert
DSGVO
Datenschutz

Verhindert, dass persönliche Daten unkontrolliert ins Internet fließen.

  • Namen, E-Mails oder IDs dürfen nicht in Suchanfragen landen
  • Du hast das Recht zu wissen, welche Daten verarbeitet werden
  • Kein Tracking der Suchhistorie ohne Einwilligung
EU AI Act
KI-Regulierung

Verlangt Transparenz, Erklärbarkeit und Fairness bei KI-Systemen.

  • Die KI muss erklären, wie sie zur Antwort kommt
  • Nutzer müssen wissen, dass sie mit einem KI-System sprechen
  • Hochrisiko-KI braucht menschliche Aufsicht

📄 Systemprompt — Technische Dokumentation

Kontext: Dieses Tool arbeitet vollständig offline. Es gibt keinen Live-API-Aufruf an ein KI-Modell. Alle Antworten sind vorskriptiert und werden per Tipp-Animation simuliert.

Simulierter Systemprompt (würde bei Online-Betrieb gesendet):

Du bist ein erfahrener Requirements Engineer und Prompt-Experte.

Aufgabe: Vergleiche den schlechten und den guten Prompt zu einem
Anwendungsfall und erkläre in 3 Sätzen, warum der gute Prompt
zu besseren KI-Ergebnissen führt.

Format:
- Satz 1: Was fehlt im schlechten Prompt (konkret benennen)
- Satz 2: Was der gute Prompt hinzufügt (Framework-Elemente)
- Satz 3: Welches Qualitätskriterium (ISO 25010 / IEEE 29148 /
  DSGVO / EU AI Act) das stützt

Eingrenzung: Keine Allgemeinaussagen. Keine Floskeln.
Jeder Satz muss testbar sein.
📚 Handbuch — Didaktische Erklärung (PromptSchule-Methodik)

Framework: RTF / PACT / CO-STAR / TRACE / RISEN / STAKE — je nach gewähltem Beispiel und Framework-Karte.

Die 5 Dimensionen im Vergleich

DimensionSchlechter PromptGuter Prompt
RolleFehlt komplettExplizit benannt (z.B. „Senior QA Engineer")
AufgabeVage AbsichtKonkrete Handlungsanweisung mit Verb
KontextKein Stakeholder, kein SystemSystem, Umgebung, Einschränkungen klar
FormatBeliebigExplizit (Gherkin, TRACE, Tabellarisch …)
EingrenzungKeine GrenzenScope, Budget, Zeitraum, Ausschlüsse

Das eine Prinzip, das hier zählt:
Ein Prompt ist eine Anforderung an die KI. Wer Requirements Engineering beherrscht — eindeutig, vollständig, testbar, konsistent (IEEE 29148) — schreibt automatisch bessere Prompts. Der Umkehrschluss gilt genauso: Wer gut promptet, versteht instinktiv, was eine gute Anforderung ausmacht.

Nachbau-Tipp:
Nimm einen eigenen schlechten Prompt aus dem Arbeitsalltag. Analysiere ihn entlang der 5 Dimensionen: Was fehlt? Welches Framework passt? Schreibe ihn neu — und teste die Antwort-Qualität. Das ist Prompt-Werkstatt in der Praxis.
